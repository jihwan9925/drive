필터적용하기

클라이언트가 보낸 요청에 대해 검토하는 기능
ex:이용권한확인, 로그인여부확인, 인코딩처리, 필요한log출력
서블릿(서비스)이용시 필요한 공통로직에 대해 설정하는 객체

필터 구현하기
1. 일반클래스를 생성하고 Filter인터페이스를 구현한다.
2. 생성한 클래스를 filter로 등록
1) web.xml방식
2) 어노테이션방식
3. 등록된 필터클래스를 어디에 적용할지 설정
1) urlpattern으로 설정
2) servletName으로 설정

doFilter메소드를 재정의해서 공통의 로직을 구현
doFilter메소드응 세개의 매개변수를 가지고 있다.
1. ServletRequest : HttpServletRequest의 부모로 request가 제공하는 메소드를 가지고 있음. HttpServletRequest로 형변환이 가능하다
2. ServletResponse : HttpServletRequest의 부모로 request가 제공하는 메소드를 가지고 있음. HttpServletRequest로 형변환이 가능하다
3. FilterChain : 다른필터 또는 연결된 서블릿을 가지고 있는 객체, 다음필터 또는 서블릿호출할때 사용한다.(doFilter()메소드호출)
